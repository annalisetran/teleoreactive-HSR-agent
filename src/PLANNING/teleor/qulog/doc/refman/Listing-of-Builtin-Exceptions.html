Appendix B Listing of Builtin Exceptions

Qulog comes with a builtin declaration for the exception type and a default type for the user exceptions that will be overwritten by a user definition: 

  def exception == builtin_exception || user_exception
  def user_exception ::= default_exception_()

The builtin_exception type is an enumeration of constructors. Below we list the constructors describing how each exception is raised.

read_term_syntax_error():

This exception is raised when there is a syntax error in reading a term via read_term

input_term_type_error(atom, string):

This exception is raised when any input is parsed but the resulting ’term’ is not a valid Qulog term. This can happen in a read, a message receive or when calling string2term.

read_op_term_syntax_error(string):

This exception is raised when there is a syntax error in reading a term via read_op_term

input_term_error(atom, string):

This exception is similar to input_term_type_error except it is raised if the read in term has an unevaluated function call.

string2term_syntax_error(string):

This exception is raised when there is a syntax error in reading a term via read_term

stream_not_writeable(string):

This exception is raised when attempting to write to a stream that is not in write mode

stream_not_readable(string):

This exception is raised when attempting to read from a stream that is not in read mode

stream_not_open(string):

This exception is raised when attempting to read from or write to a stream that is not open

action_failure(string):

This exception is raised when an called action fails in the body of the chosen rule

no_matching_action_rule(string):

This exception is raised when an call to an action does not match any rule head and guard

no_matching_function_rule(string):

The same as above for function calls

no_matching_tr_rule(tel_term):

The same as above for teleo procedure calls

arithmetic_function_domain_error(term):

This exception is raised when the arguments to an arithmetic function are not in the domain of the function such as when dividing by zero

remote_query_ontology_mismatch(string):

This exception is raised on the client side when some term within a remote query from a client is not of type term within the server. Since the term is a valid term on the client side then this must be because there is a mismatch in the type declarations in the server

remote_query_exception(string):

This exception is raised on the client side when an exception is raised on the server side - the server side exception is turned into a string and that becomes the argument of the client side exception.

pedro_not_connected(atom):

This exception is raised when we attempt to interact with Pedro but we have not connected to Pedro

pedro_name_already_registered(atom):

This exception is raised when we attempt to register with Pedro using a name that has already been registered

fork_thread_name_exists(atom):

This exception is raised when we attempt to fork a thread with a name that is already the name of a already forked thread

qp_exception(string):

This exception is raised when some code raises an exception at the QuProlog level. The QuProlog exception is converted to a stringas the argument of the exception

qulog2qp_type_exception(string):

This exception is raised when output arguments of interface code are not of the required type (see the next section)

no_agent_to_kill():

This exception is raised when the argument to kill_agent is not the name of a current agent

no_matching_task_to_kill(atom):

This exception is raised when the argument to kill_task is not the name of a current task

not_start_task_TR_procedure(string):

This exception is raised when we try to start a task that has not been declared as a start teleo procedure

task_name_in_use(atom):

This exception is raised when we try to start a task using the name of current task
